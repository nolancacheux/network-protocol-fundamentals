Résolution d'adresse - Protocole ARP

Cours  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           


Le protocole ARP a été conçu pour assurer la résolution d'adresse Ethernet.

Il assure dynamiquement la correspondance entre les adresses IP et Ethernet. Cette correspondance sera maintenue, même en cas de modification de l'architecture ou de déplacement d'équipement : à chaque adresse IP correspond une adresse Ethernet.

Lorsqu'un poste souhaite envoyer un message à un autre poste situé sur un même réseau local, il doit préalablement avoir connaissance de l'adresse IP de son correspondant.

Le cas d'un poste situé sur un autre réseau local sera étudié ultérieurement.

Compte tenu qu'il ne dispose pas de l'adresse Ethernet de son correspondant, il sollicitera tous les postes de ce même réseau local afin que le poste concerné lui fasse parvenir son adresse Ethernet.

Question
 Quel est le type d'adresse Ethernet utilisé par le protocole ARP pour réaliser la requête de résolution d'adresse ?
Unicast.
Cette option nécessiterait d’envoyer autant de trames de requête qu’il y aurait de postes à interroger.
Multicast.
Cette option permettrait de réduire le trafic à l’envoi d’une seule trame de requête mais ne concernerait que les postes écoutant sur cette adresse multicast.
Broadcast.
Cette option permet de réduire le trafic à l’envoi d’une seule trame de requête.
Aucune de ces propositions n'est satisfaisante.
Reprenez le cours, cette notion n’est pas bien comprise.

Question
 Est-ce que la résolution d'adresse peut être étendue pour interroger un poste d'un autre réseau local ?
Oui
Le périmètre d’utilisation d’une adresse Ethernet se limite au réseau local où elle est située. Une requête ARP n’a donc pas pour objet d’être propagée en dehors du réseau local depuis lequel elle a été envoyée.

Question
 Dans le cadre d'une requête ARP, est-ce que la valeur de l'adresse destinataire située dans l'entête du protocole Ethernet a la même valeur que celle située dans la trame du protocole ARP ?
Oui
D’une part, on y trouve une adresse de broadcast pour adresser tous les destinataires, d’autre part une valeur "nulle" compte tenu que l’adresse de l’interlocuteur n’est pas encore connue.
Non
Les fonctions de ces deux champs ne sont pas identiques même s’il s’agit d’adresses Ethernet de destinataires.
