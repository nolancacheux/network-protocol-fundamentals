Cache ARP - Protocole ARP

Cours  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           


La dynamique du protocole ARP assure une mise à jour régulière des informations détenues par les autres postes.

Ce trafic ne serait plus négligeable si ces informations n'étaient pas conservées localement mais redemandées à chaque émission de message.

Le cache ARP apporte la solution en conservant les informations collectées.



Cette conservation locale se fait pour un temps limité afin de ne pas surcharger le réseau local tout en garantissant un état cohérent avec la réalité du terrain.

Question
 Que se passe-t-il si un poste est remplacé par un autre en gardant la même adresse IP ?
Un poste n'ayant jamais eu d'échange avec celui remplacé, n'aura aucune difficulté pour communiquer avec le remplaçant.
Le cache ARP n’étant pas renseigné, la mise à jour est immédiate.
Un poste ayant eu des échanges récents avec celui remplacé, ne pourra plus jamais communiquer avec le remplaçant.
Le contenu du cache ARP deviendra obsolète, et le processus de mise à jour se déclenchera. Pendant cet intervalle, les échanges ne pourront plus avoir lieu.
Un poste ayant eu des échanges récents avec celui remplacé, tentera de communiquer sans succès avec le remplaçant pendant un temps limité au delà duquel les échanges pourront de nouveau avoir lieu.
