Histoire - Protocole Ethernet

Invention du protocole Ethernet  
Compréhension :   0   0   0   0   0   0     QCM :   0   0   0           
Comme dans le cadre de ce qui vous a été présenté, les concepteurs souhaitaient mettre en oeuvre un protocole de communication simple.




David Redell (Xerox), Rich Seifert (DEC) et Rob Ryan (Intel) ont créé la version 1.0 de la spécification Ethernet en 1980, et Bob Printis (Xerox) a représenté Ethernet auprès de l'organisme de normalisation IEEE pour créer Ethernet IEEE 802.3.

Près de 50 ans plus tard, la seconde version de ce protocole, Ethernet II, assure la connexion d'une très grande majorité d'équipements sur des réseaux locaux.

Question
 Quelles sont les caractéristiques du protocole Ethernet 802.3 ?
Il ne permet pas à lui seul d’identifier l'origine des informations émises.
L’entête comporte l’adresse de source qui correspond à l’adresse de la carte réseau de l’expéditeur des données.
Il ne dispose pas de champ permettant d'identifier les applications source et destination.
L’entête comprend uniquement les adresses de source et de destination, et une information sur le nombre d’octets de données.
Il permet de transporter des données brutes sans se soucier des applications concernées par celles-ci.

Sa structure permet d’encapsuler un entête supplémentaire 802.2 pour identifier les applications source et destination.
Le champ donnée, appelée aussi payload, permet d’encapsuler tout type d’informations, des données issues d’une application mais aussi des données de "service" telles qu’un entête 802.2. Dans ce cas, les systèmes doivent utiliser la même structure de trame.
Associé au protocole 802.2, il est plus performant que le protocole Ethernet II, notamment en permettant de différencier davantage d’applications source et destination.
Il n’y a aucune différence de performance, il s’agit d’un autre structure de trame offrant des fonctionnalités identiques.
